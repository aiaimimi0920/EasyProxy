# MiSub 节点净化管道指南

当前仓库中的 MiSub 已经内置一套可视化的节点净化管道，用于在输出订阅前统一处理节点名称、重复节点和排序规则。

这套能力在 UI 中主要对应两个入口：

- `全局设置 -> 节点净化管道`
- `订阅组 -> 高级设置 -> 节点净化配置来源`

如果订阅组没有启用自定义净化配置，则默认继承全局设置。

## 1. 执行顺序

当前分支的净化管道执行顺序是固定的：

1. 正则重命名
2. 智能去重
3. 模板重命名
4. 排序

这意味着：

- 正则规则适合先把杂乱命名统一。
- 去重会基于重命名前后已经解析出的节点元信息工作。
- 模板重命名适合最后生成统一的展示名称。
- 排序永远发生在最终输出前。

## 2. 当前支持的能力

### 2.1 正则重命名

适用于把机场原始命名快速规整成统一风格。

常见场景：

- 把 `Hong Kong` 统一改成 `HK`
- 去掉 `IPLC`、`专线`、`倍率` 等噪声标签
- 合并机场命名中的多余空格和重复前缀

建议把“清洗噪声”的规则放在前面，把“格式统一”的规则放在后面。

### 2.2 智能去重

当前支持两类去重模式：

- `url`
- `serverPort`

推荐默认使用 `serverPort`，这样可以把同一落点的重复节点合并掉。  
如果你希望不同协议保留并行副本，可以打开 `includeProtocol`。

在 `prefer.protocolOrder` 中可以定义“相同落点时优先保留哪种协议”。默认顺序偏向：

`vless -> trojan -> vmess -> hysteria2 -> ss -> ssr`

### 2.3 模板重命名

模板重命名适合生成统一、可读、可排序的节点名称。

默认模板示例：

```text
{emoji}{region}-{protocol}-{index}
```

常用变量：

- `{emoji}`：地区旗帜
- `{region}`：地区代码，如 `HK`、`US`
- `{region:zh}`：中文地区名，如 `香港`、`美国`
- `{protocol}`：协议名
- `{index}`：按作用域递增的编号
- `{name}`：原始名称
- `{server}`：服务器地址
- `{port}`：端口

常见模板示例：

```text
{emoji}{region}-{protocol}-{index}
{region:zh}-{name}
{emoji}{region}-{protocol}-{port}
```

### 2.4 排序

当前支持按以下常见字段排序：

- `region`
- `protocol`
- `name`
- `server`
- `port`

推荐的默认排序思路：

1. 先按地区
2. 再按协议
3. 最后按名称

如果你只想按端口或服务器排序，也可以只保留单一排序键。

## 3. 全局与订阅组的关系

### 3.1 全局默认值

`全局设置` 中的净化配置会写入 `defaultNodeTransform`。  
当某个订阅组没有单独配置时，就会继承这份默认规则。

### 3.2 订阅组覆盖

在订阅组高级设置中，可以切换：

- `使用全局设置`
- `自定义`

切换到 `自定义` 后，当前订阅组会保存自己的 `nodeTransform`，不再跟随全局默认值变化。

## 4. 与前缀设置的关系

当前仓库同时保留了“前缀设置”和“节点净化管道”两条能力线：

- 前缀设置负责在输出前给手动节点或订阅节点附加基础前缀
- 净化管道负责在最终输出阶段统一改名、去重和排序

需要注意：

- 当你启用了模板重命名时，最终名称往往会由模板完全接管。
- `分组名称前缀` 这类旧式前缀逻辑不适合和强模板同时使用。
- 如果你的目标是生成稳定、机器友好的命名，优先使用模板重命名。

## 5. 推荐配置

### 5.1 稳定分享型配置

适合公开展示或长期分享：

- 正则重命名：清理噪声关键字
- 去重：`serverPort`
- 模板：`{emoji}{region}-{protocol}-{index}`
- 排序：地区 -> 协议 -> 名称

### 5.2 保留机场原味配置

适合只做轻量清洗：

- 正则重命名：只删噪声词
- 去重：关闭或仅 `url`
- 模板：关闭
- 排序：仅按名称

## 6. 当前分支的边界

与上游较新的“完整操作符链”相比，当前仓库这一分支更偏向稳定的命名管道，重点是：

- 正则重命名
- 模板重命名
- 去重
- 排序

当前并未在这条分支中引入更激进的筛选脚本运行时，因此如果你需要做复杂筛选，建议优先从：

- 订阅源选择
- 手动节点分组
- profile 组合范围

这几层先收敛输入，再用净化管道做统一输出。
